      • aggregateEvents

        public javax.ws.rs.core.Response aggregateEvents(EventViews.Aggregation aggregation)
                                          throws BadRequester
        Aggregate of global events. Standard list parameters for paging and filtering apply. In this case the stream is grouped according to one of two criteria. The default groups all sequential events that have the same actioner.

        An alternate strict aggregation stategy groups items that are:
        1. triggered by the same actioner
        2. affecting on the same subjects
        3. have the same scope
        4. have the same log message

        Alternately, aggregation can be disabled by using the off parameter value.

        Parameters:
        aggregation - Aggregate events by strict, or user
        Throws:
        BadRequester
