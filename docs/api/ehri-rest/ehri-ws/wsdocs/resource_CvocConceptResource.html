CvocConceptResource

Provides a web service interface for the Concept model. Note that the concept creation endpoint is part of the VocabularyResource and creation without a Vocabulary is not possible via this API.

The following resources are applicable:

  • /classes/CvocConcept
  • /classes/CvocConcept/{id}
  • /classes/CvocConcept/{id}/broader/list
  • /classes/CvocConcept/{id}/CvocConcept
  • /classes/CvocConcept/{id}/list
  • /classes/CvocConcept/{id}/narrower/{idNarrower}
  • /classes/CvocConcept/{id}/related/{idRelated}
  • /classes/CvocConcept/{id}/related/list
  • /classes/CvocConcept/{id}/relatedBy/list

/classes/CvocConcept

Mount Point: /classes/CvocConcept

GET

List available resources. The default behaviour is to return a page of 20 items.

The default list limit can be overridden using the limit parameter, and disabled completely using a value of -1.

Item lists are paginated using the page parameter.

The page, limit, and total number of items is returned in the Content-Range response header in the form: 

page=1; count=20; total=50

If the header X-Stream is set to true no count of total items will be performed. This is more efficient when returning large numbers of items or when limiting is disabled completely.

Example: 

{@code
curl http://localhost:7474/ehri/[RESOURCE]/list?page=5&limit=10
}

Response Body

element: (custom)
media types: application/json

A list of serialized item representations
