POST /tools/regenerate-id/{id}

Regenerate the hierarchical graph ID for a given item, optionally renaming it.

The default mode is to output items whose IDs would change, without actually changing them. The collisions parameter will only output items that would cause collisions if renamed, whereas tolerant mode will skip them altogether.

The commit flag will cause renaming to take place.

Request Parameters
name type description default constraints
id path the item's existing ID   regex: [^/]+
collisions query only output items that if renamed to the regenerated ID would cause collisions false boolean
commit query whether or not to rename the item false boolean
tolerant query skip items that could cause collisions rather then throwing an error false boolean
Response Body
media type data type description
text/csv (custom) a tab old-to-new mapping, or an empty body if nothing was changed
