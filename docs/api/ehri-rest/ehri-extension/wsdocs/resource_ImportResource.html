/import/ead

Mount Point: /import/ead

POST

Import a set of EAD files. The POST body can be one of:

  • a single EAD file
  • multiple EAD files in an archive
  • a plain test file containing local file paths
The Content-Type header is used to distinguish the contents.
Note: The archive does not currently support compression.

The way you would run with would typically be: 

{@code
curl -X POST \
-H "X-User: mike" \
--data-binary @ead-list.txt \
"http://localhost:7474/ehri/import/ead?scope=my-repo-id&log=testing&tolerant=true"

# NB: Data is sent using --data-binary to preserve line-breaks - otherwise
# it needs url encoding.
}

(Assuming ead-list.txt is a list of newline separated EAD file paths.)

(TODO: Might be better to use a different way of encoding the local file paths...)

Parameters

name description type default
scope The id of the import scope (i.e. repository) query
tolerant Whether or not to die on the first validation error query false
log Log message for import. If this refers to an accessible local file its contents will be used. query
properties A local file path pointing to an import properties configuration file. query
handler The fully-qualified handler class name (defaults to EadHandler) query
importer The fully-qualified import class name (defaults to EadImporter) query

Request Body

element: data
media types: application/xml
text/plain
text/xml
application/octet-stream

File data containing one of: a single EAD file, multiple EAD files in an archive, a list of local file paths. The Content-Type header is used to distinguish the contents.

Response Body

element: (custom)
media types: application/json

A JSON object showing how many records were created, updated, or unchanged.

/import/skos

Mount Point: /import/skos

POST

Import a SKOS file, of varying formats, as specified by the "language" column of the file extensions table here.

Example: 

{@code
curl -X POST \
-H "X-User: mike" \
--data-binary @skos-data.rdf \
"http://localhost:7474/ehri/import/skos?scope=gb-my-vocabulary&log=testing&tolerant=true"
}

Parameters

name description type default
scope The id of the import scope (i.e. repository) query
tolerant Whether or not to die on the first validation error query false
log Log message for import. If this refers to an accessible local file its contents will be used. query
format The RDF format of the POSTed data query

Request Body

element: stream
media types: */*
application/xml

A stream of SKOS data in a valid format.
